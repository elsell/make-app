# Security policy

Report suspected vulnerabilities privately through this repository's GitHub
security advisory form. Do not publish exploit details, credentials, personal
data, or unredacted logs in an issue.

Supported application versions and backport policy must be declared before the
first public release. Until then, only the current default branch is supported.
Include the application version, make-app generator version and template schema
from `.make-app.json`, host OS, affected boundary, reproduction steps, and impact.
