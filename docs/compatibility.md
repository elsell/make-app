# Compatibility and stability

`make-app` is an alpha-quality `v0.x` tool: commands and generated layouts may
change between minor releases. Generated applications own their code and do not
receive changes automatically.

| Generator release | Template schema | Mutation support |
| --- | ---: | --- |
| `v0.4.3` | 4 | Current supported schema-4 generator release <!-- make-app:release-version:start --> |
| Current `main` | 4 | Same persisted schema contract as `v0.4.3`; unreleased behavior may differ <!-- make-app:release-version:end --> |
| `v0.2.0`–`v0.4.0` | 4 | Existing schema-4 projects; newer releases add adoption, native delivery, and hardened container improvements |
| Earlier `v0.x` | 1–3 | Upgrade documentation only; mutation commands fail closed |

Schema 3 projects must complete the documented v3-to-v4 repository upgrade
before using current mutation commands. A schema number alone is not a migration.
Native mobile build guarantees apply only to projects generated by a release
whose compatibility row explicitly includes them.
